補充：建立虛擬網路交換器（橋接器）

為了確保 SQL 伺服器能夠在虛擬機之間進行有效的連線，必須先在 Hyper-V 中建立虛擬網路交換器（橋接器）。

建立虛擬交換器的步驟：

  1. 開啟 Hyper-V 管理員：

    在 Windows 搜尋欄中輸入 “Hyper-V 管理員” 並開啟應用程式。

  2. 建立虛擬交換器：

    在 Hyper-V 管理員的右側面板中，選擇 “虛擬交換器管理員”。

    點擊 “新建虛擬交換器” 按鈕，並選擇 “外部” 類型的交換器。

  3. 配置虛擬交換器：

    為虛擬交換器指定一個名稱，例如 “BridgeSwitch”。

    在 “連線類型” 中，選擇您的實體網路介面卡（NIC），這樣虛擬機器才能與外部網絡通信。

    勾選 “允許管理作業系統共用此網路介面卡” 以便主機和虛擬機器共用網路連線。

    完成配置後，點擊 “確定” 以保存設定。

補充說明：WAN（廣域網路）是指在地理上分散的網路系統，用於連接不同地點的網路設備。

特性 第 1 代虛擬機器 第 2 代虛擬機器
支援作業系統 較舊的作業系統：例如 Windows 7、Windows Server 2008 等。 較新的作業系統：例如 Windows 10、Windows Server 2016 及更新版本。

硬體支援 支援舊版硬體：例如舊版的網路設備。 較高的效能：支援更多的虛擬 RAM 和虛擬 CPU。

第五步：連接/建立虛擬硬碟

這邊以自己需求為主，可以自行建立也可掛載提前設定好的虛擬硬碟。

補充：虛擬硬碟 (VHD/VHDX)

虛擬硬碟 (VHD) 是一種磁碟映像檔案格式，具有與實體硬碟類似的功能，主要設計用於虛擬機器。它可以在一個實體硬碟上模擬多個虛擬硬碟，基本用來安裝作業系統或存儲資料 ([微軟介紹)](https://learn.microsoft.com/zh-tw/windows-server/storage/disk-management/manage-virtual-hard-disks))。

如何建立虛擬硬碟？

  1. 開啟磁碟管理：
    • 在 Windows 搜尋欄中輸入 “電腦管理” 並開啟它。
    • 在左側選單中選擇 “磁碟管理”。
  2. 建立虛擬硬碟 (VHD)：
    • 在 “動作” 功能表上，選取 “建立 VHD”。
    • 在 “建立並連結虛擬硬碟” 對話方塊中，指定您想要在實體電腦上儲存 VHD 檔案的位置。
    • 在 “檔案名稱” 欄位中，輸入磁碟機的名稱。
    • 從 “另存新類型” 下拉式功能表中選取 “虛擬磁碟檔案 (*.vhd)”。
    • 選取 “儲存”。
  3. 設定虛擬硬碟大小和類型：
    • 在 “虛擬硬碟大小” 底下輸入檔案大小。
    • 在 “虛擬硬碟類型” 下，選取 “動態擴充” 或 “固定大小”。
    • 選取 “確定” 來完成。
  4. 連接虛擬硬碟：
    • 在 “磁碟管理” 中，選取 “連結 VHD”。
    • 在 “連結虛擬硬碟” 對話方塊中，選取 “瀏覽” 按鈕，然後尋找 .vhd 檔案。
    • 選取 “確定”。

微軟複雜性密碼規範

在進入伺服器之後首先會需要你創建密碼，而微軟的伺服器的設定必須符合微軟的複雜性需求。

微軟的密碼複雜性需求旨在提高密碼的安全性，防止未經授權的訪問。以下是主要的要求：

  • 密碼長度：至少六個字元
  • 字元類型：密碼必須包含以下四類字元中的至少三種：
    • 英文大寫字母 (A 到 Z)
    • 英文小寫字母 (a 到 z)
    • 基底 10 個數字 (0 到 9)
    • 非英數字元 (例如 !、$、#、%)
  • 帳戶名限制：密碼不得包含三個或三個以上來自使用者帳戶名中的字元

這邊使用舉例：!qaz@WSX

定序介紹

定序 指的是排序的方式。在資料庫或其他系統中，定序會影響資料的排列順序。常見的定序包括：

  • 英文定序：A~Z
  • 數字定序：1~10

在安裝過程中，通常可以選擇預設的定序值。資料庫中可以根據需要自定義定序。例如，使用筆畫數（Stroke）來進行排序。

注意：如果選擇勾選「區分大小寫」，則小寫字母 a 與大寫字母 A 會被視為不同的字符。類似地，中文全形和半形字符也會被區分對待。

定序的選擇會影響到資料的排序和比較方式，因此在設定時需要根據實際需求進行選擇。
